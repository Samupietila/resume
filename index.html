Healthcare Center Queue Simulation

The purpose of this project is to demonstrate the student's object-oriented programming skills in the Java programming language by simulating the behavior and possible formation of queues in a hospital. The project varies the formation and behavior of queues by changing the density of customer arrival times and the service times of the service points. Queue formation is simulated using the FIFO (First In, First Out) principle and a priority queue. Each customer receives a random arrival time and service time.

Features:

  • Simulation of queue formation in a hospital setting
  • Variation of customer arrival times and service times
  • Implementation of FIFO principle and priority queue
  • Random assignment of arrival and service times for customers
  • Calculation of average queue lengths and customer queuing times
  • Measurement of service points' service times
  • Determination of the time customers spend in the hospital
  • Comparison of different queue behaviors and formations
  • Optimization of service points based on customer density

Technology: The project is implemented using Java, showcasing object-oriented programming techniques.

Simulation Metrics: The simulation calculates averages for each queue length, customer queuing time, service points' service times, and the time customers spend in the hospital. These calculations are based on each customer's arrival and departure times, service points' service times, and queuing times, as well as the number of customers.

User Interface: The simulation's user interface is developed using JavaFX, providing an interactive and visual representation of the queue behavior and formation.

HearBeatMonitor

The HearBeatMonitor project focuses on developing a heart rate monitor suitable for home and office use. The primary objective was for students to gain hands-on experience with configuring and utilizing devices like the Raspberry Pico microcontroller and a Wi-Fi router. The project entailed using sensors and other accessories with the microcontroller, developing an embedded system with MicroPython, and writing algorithms to detect heart rate and its variability. Additionally, it involved setting up a server, establishing a wireless link between the server and the microcontroller, reading and interpreting hardware specifications, using GitLab, converting analog signals to digital, leveraging external cloud services, and creating a user-friendly, bug-free product for end-users.

Features:

  • Configuration and use of Raspberry Pico microcontroller
  • Integration with Wi-Fi router
  • Use of sensors and accessories with the microcontroller
  • Embedded system development with MicroPython
  • Algorithm development for heart rate detection and variability
  • Server setup and wireless communication
  • Reading and interpreting hardware specifications
  • Use of GitLab for version control
  • Analog to digital signal conversion
  • Integration with external cloud services
  • Development of a user-friendly, bug-free product

Technology: The project is implemented using MicroPython for the embedded system.

Cloud Service: The HearBeatMonitor uses a cloud service to calculate and present Heart Rate Variation (HRV) data based on photoplethysmographically (PPG) measured heart rate values. This allows end-users to effectively monitor their health.

Features:
