Healthcare center queue simulation

The purpose of this project is to demonstrate the student's object-oriented programming skills in the JAVA programming language by simulating the behavior and possible formation of queues in a hospital. In the project, the formation and behavior of queues are varied by changing the density of customer arrival times and service times of the service points. Queue formation is simulated using the FIFO (First in, first out) principle and a priority queue. Each customer receives an arrival time and service time randomly. During the simulation, averages are calculated for each queue length, customer queuing time, service points service time, and the time customers spend in the hospital using each customer's arrival time and departure time, service points service times and queuing times, and the number of customers. With the calculated values, behavior of different queues and formation can be compared, and based on them, it can be determined what number of service points is optimal with each customer density.

HearBeatMonitor

The topic of the project was a heart rate monitor for home and office use. The project was done mainly for the students to learn about configuring and using devices such as Raspberry Pico microcontroller and a Wi-Fi-router, using sensors and other accessories in connection with microcontroller, developing an embedded system using MicroPython and writing an algorithm to detect heart rate and its variability, setting up a server and establishing a wireless link between said server and microcontroller, knowing how to read and interpret specifications for hardware devices, using GitLab, converting analog signals to digital, using external cloud services and establishing a user-friendly, bug-free product for end-users. The goal of the project was to create a heart rate monitor device with an embedded system that uses a cloud service to calculate and present Heart Rate Variation (HRV) data based on photoplethysmographically (PPG) measured heart rate values for an end-user to monitor their health. It was the final assignment of the Hardware 2 course.

Tennis Betting Application

The Tennis Betting Application is an interactive web application that allows users to place bets on virtual tennis tournaments. Designed for both new and experienced users, the application features user registration, tournament selection, and match betting.

Features:

  • User registration and login
  • Selection of tournament, match, and player
  • Ability to place bets on matches
  • Management and display of player information
  • User guides and support

Technology: The project is implemented using HTML, CSS, and JavaScript.

Database: The Tennis Betting Application uses an SQL database to store and manage user data, match results, and betting rates. The database includes the 2017 men's ATP results and rates, ensuring data reliability and efficient handling.
